React Dashboard Application

Hooks & Refs | Context API | Syncfusion

Fully mobile responsive and customizable React Dashboard Application where advanced best React practices such as: Folder and File structure, Hooks and Refs are used. There are many different components in many different pages in this app and a general possibility of scalling smaller React Apps to a higher enterprise level size can be achieved with this project. This Admin Dashboard is fully mobile responsive, which makes it feel like a native mobile application.

This Admin Panel includes one dashboard, three pages, four apps and seven fully functional charts. The Orders Page includes a pagination system, the Employees table includes a completely functional search and filtering functionality, while the Customers user table is fully editable, which means you can select multiple rows and delete them, as well as edit each view individually. Sidebar can also be collapsed to view the contents in its entirety.

The dashboard has a beautiful view of all the most important metrics such as; earning, recent transactions, sales overview and more. There are 6 beautiful theme colors as well as toggle between the dark and light modes

Below are four major apps that are crucial to almost any dashboard included in this project:

  • First App is a fully functioning calendar - you can drag n drop events, and edit the title, locations, start and end dates and every single other little detail The calendar view can be changed from Day, Week, Month and even an Agenda list
  • The 2nd App is a Kanbn Board, just like an entire Jira app right inside of our dashboard
  • Third App is a fully-fledged wysiwyg editor, you can make texts bold, add images and essentially modify the texts in anyway you can think of
  • The fourth is a Color Picker App, highly useful if you want to allow your user to choose specific colors, for example to theme their applications

There are 7 different kinds of charts (Line, Area, Bar, Pie, Financial, Color-Mapping and Pyramid Charts). Additional widgets like a Shopping Carts, the Chat, Notifications and a User Profile are also embedded.


Advanced state of the entire application is managed using React Context API. Mmost importantly, i used Syncfusion - the world's best UI component for building powerful web, desktop and mobile applications


This is not a specialized dashboard, which means that each piece of it can be used for absolutely any app.

Job/Gigs

Laravel | Breeze | Queued Mail Notification

This project is a fully functioning job gigs site designed with Laravel. I use migrations and factories to create and seed the database. The breeze package was installed and configured for authentication to developed both a landing page and an individual listing page with the help of blade components and tailwind css which enable end users to register, create listings to be displayed on the job board. When Gigs are created or deleted, job notification is queued and an email alert is sent to an administrator. Laravel resource routing was used to assign the typical CRUD operations (create, read, update, and delete) to a controller with a single line of code Resource Routes. For implicit binding, i used Route Model Bindings to automatically inject the model instances directly into the routes.
